# 跑团Replay动态漫画桌面应用 — 产品说明

## 1. 产品概述

本应用是一款桌面端 TRPG（桌上角色扮演游戏）Replay 动态漫画创作工具。它将**漫画排版、动画编排、AI 素材生成、视频导出**整合到一个创作环境中，面向个人创作者使用。

### 核心工作流

```
海豹骰 Log (docx)  ──→  Log 分屏查看  ──→  复制文本到气泡
                                               │
AI 生成 / 本地素材  ──→  素材库管理    ──→  拖入画布编辑
                                               │
三层图层 + 时间线   ──→  动画编排      ──→  实时预览
                                               │
                                          FFmpeg 导出 MP4
```

### 目标用户

- TRPG 主播/UP主：制作跑团 Replay 视频内容
- 同人创作者：给跑团记录配图+配音
- 跑团 GM：为团友制作回顾纪念

## 2. 功能模块

### 2.1 制作面（核心创作区）

采用三层图层架构，配合时间线动画系统。

**三层图层架构：**

| 层级 | 名称 | 内容 |
|------|------|------|
| 底层 | 漫画框层 | 分镜边框、背景图、场景设定 |
| 中层 | 人物层 | 角色立绘、怪物、NPC 图像 |
| 顶层 | 气泡层 | 对话气泡、旁白框、文字展示 |

所有层级均支持独立的图片插入与混合模式。

**动画效果清单：**

| 效果 | 说明 | 有无结束时间 |
|------|------|:---:|
| 平滑移动 | 起点→终点坐标过渡，速度曲线可调 | ✅ |
| 淡化 | 透明度渐变，可设置自动还原 | ✅ |
| 缩放 | 放大/缩小平滑过渡 | ✅ |
| 果冻/呼吸 | 轻微起伏营造呼吸感或晃动 | ❌ 持续 |
| 材质覆盖 | 保留轮廓，叠加颜色/纹理 | ✅ |
| 帧动画 | 多帧序列播放，可循环 | ✅ |
| UT文字动画 | 逐字显示/全显，可选短音频绑定 | ✅ |

**效果叠加**：多个效果在同一时间段可叠加，产生复合动画。如平滑移动 + 缩放 + 淡化 = 角色"边走远边消失"。

**时间线系统**：每个图像拥有独立时间线轨道，类似视频剪辑软件。有结束时间的效果以"效果块"形式在轨道上呈现，可拖拽调整起止位置。无结束时间的效果作为图像的持续属性。

**内置播放逻辑**：按图层顺序（底→中→顶）播放，层内图像可选"逐个显现"或"一次全部显现"。全部图像完成动画后经过静默保持时长自动进入下一页。

**缩放/平移**：支持画布缩放（滚轮）与平移（空格+拖拽），方便在大画布上精细操作。

### 2.2 对话气泡系统

- 气泡框与文字分离：框体可变形的 Mesh 网格，文字不参与变形
- 9 个控制点：四角 + 四边中点 + 中心，拖拽可改变气泡形状
- 文字感知边框变化自动重排（换行适配）
- 可配置样式：背景色、边框色、边框粗细、圆角、内边距
- 可选尾巴指针：指向说话角色

### 2.3 Log 展示

- 制作面右侧 30% 分屏展示跑团 Log
- 解析海豹骰服务器导出的 `.docx` 格式 Log
- 保留原文本字体颜色与格式（GM 红色、玩家蓝色等）
- 支持选中复制 → 粘贴到气泡文本

### 2.4 素材库

- 右侧树形文件管理器，支持多级文件夹分类
- 管理图片（PNG/JPG/WEBP）、音频（MP3/WAV/OGG）、动画预设
- **动画预设系统**：将配置好动画效果的图像保存为预设，再次调用时自动继承所有时间线、效果块与参数
- 从素材库直接拖入制作面放置
- 支持名称搜索与类型筛选

### 2.5 AI 素材生成

- 集成 ComfyUI 本地引擎（主）与云端 API（备）
- 多智能体画风管理：每个 agent 绑定一组参考图与提示词模板
- LoRA 微调保角色一致性：上传 5-15 张角色图自动训练
- IP-Adapter 免训练方案：单张参考图快速生成
- 支持生成角色、背景、道具等多品类素材
- 生成结果可直接保存至素材库

### 2.6 预览与导出

- 实时预览：在画布中播放完整动态漫画
- 页面切换：支持上一页/下一页/跳转
- 视频导出：FFmpeg 帧序列合成，一键导出 MP4
- 导出参数：分辨率预设（1920×1080, 1280×720）+ 帧率（24/30/60）+ 质量

### 2.7 项目管理系统

- 文件夹式项目：project.json + assets/ + exports/
- JSON 序列化 + schemaVersion 版本号
- 自动保存（debounce 30 秒）
- 无限撤销/重做（Ctrl+Z/Ctrl+Y，最大深度 200）

## 3. 与原始文档的差异说明

以下是设计过程中与原始 [产品初步文档说明.md](../产品初步文档说明.md) 的主要调整：

| 项目 | 原始方案 | 最终方案 | 原因 |
|------|---------|---------|------|
| 气泡文字变形 | 字体跟随网格同步形变 | 文字不参与变形，只做重排 | 技术实现复杂度高，可读性差 |
| Canvas 库 | PixiJS + Fabric.js/Konva.js | 仅 PixiJS v8 | 避免双库事件冲突和坐标同步问题 |
| 目标用户 | 未明确 | 个人使用 | 影响所有技术决策 |
| 存储格式 | 未定义 | 文件夹 + project.json | 素材与数据分离，git 友好 |
| 状态管理 | 未定义 | Zustand（React + PixiJS 共享） | 两个渲染世界的唯一数据桥梁 |
