# 开发计划

## 总体策略

- **增量可运行**：每个 Phase 结束都有可运行、可操作的产出物
- **可扩展优先**：效果插件系统从一开始就按接口设计，后续添加效果无需改动核心引擎
- **核心先行**：先做渲染和编辑器，再做 AI 集成（避免 AI 依赖阻塞核心体验）

## Phase 0：技术验证（预计 1-2 周）

**目标**：验证核心技术方案可行

- [ ] PixiJS 三层渲染原型（静态图片叠加 + 混合模式）
- [ ] 气泡 Mesh 形变可行性（SimpleMesh + 控制点拖拽）
- [ ] ComfyUI JSON API 连通性测试（本地或远程）
- [ ] FFmpeg 管道输入帧数据 → MP4 测试
- [ ] docx 解析（mammoth）验证——海豹骰导出的实际文件测试
- [ ] 项目 JSON Schema 第一版设计并验证（手写 sample project.json 反序列化）

**交付物**：技术验证报告 + 各验证点的 Demo 代码片段

## Phase 1：骨架搭建 + 静态渲染（预计 2-3 周）

**目标**：能启动应用，看到面板布局，能在画布上放图片并做基础变换

```
┌─────────────┬──────────────────────────┬─────────────┐
│   素材库     │       Canvas 画布         │  属性面板    │
│  (树形管理)  │                          │  (选中属性)  │
│             │   ┌──────────────────┐   │             │
│  📁 角色    │   │                  │   │  x: 200     │
│  📁 背景    │   │   [渲染区域]      │   │  y: 150     │
│  📁 道具    │   │                  │   │  w: 300     │
│  📁 音频    │   └──────────────────┘   │  opacity: 1 │
│             │                          │             │
├─────────────┴──────────────────────────┴─────────────┤
│               时间线面板 (占位)                        │
└──────────────────────────────────────────────────────┘
```

- Electron + React + Vite 脚手架搭建
- 面板布局（左素材库 / 中画布 / 右属性 / 下时间线占位）
- PixiJS Application 初始化（黑色画布）
- 从素材库拖图片到画布
- 图片选中、拖拽移动、角手柄缩放、旋转
- 属性面板与选中图片同步
- 图层可见性控制

**交付物**：可运行的应用骨架，能在画布上放图并操作

## Phase 2：效果系统 + 时间线（预计 3-4 周）

**目标**：能为图片添加动画效果，通过时间线编辑，并播放动画

- EffectPlugin 接口定义 + 注册表
- 效果叠加计算引擎
- 内置效果实现：Move、Fade、Scale（首批三个）
- 内置效果实现：Wobble、Material、FrameAnimation、UTText
- 时间线 UI（标尺、轨道、播放头）
- 每个 Item 独立轨道行
- 效果块渲染、拖拽、调整时长
- 播放控制（Play/Pause/Stop/Scrub）
- 播放调度器（层顺序 + sequential/parallel 模式 + 静默保持）

**交付物**：能用时间线编排动画并播放的 MVP

## Phase 3：气泡 + Log + 项目系统（预计 2-3 周）

**目标**：完整的漫画编辑体验

- 气泡编辑器（Mesh 变形 + 文本自适应 + 样式属性）
- Log 查看器（docx 解析 + 富文本显示 + 复制粘贴）
- 项目创建/打开/保存/另存为
- JSON 序列化/反序列化 + schemaVersion 迁移
- 自动保存（debounce 30s）
- 撤销/重做栈（Ctrl+Z/Y，深度 200）

**交付物**：能完整编辑一页漫画、管理项目的产品

## Phase 4：AI 集成（预计 2-3 周）

**目标**：从 AI 生成素材并融入工作流

- ComfyUI 进程管理（spawn/stop/health-check）
- 工作流 JSON 构建器 + 任务提交/轮询
- AI 生图面板 UI
- 多智能体画风管理
- IP-Adapter 快速生成
- LoRA 训练触发（基础版，后续增强）
- 云端 API 降级方案

**交付物**：能从 AI 生成图片并保存到素材库

## Phase 5：预览 + 导出 + 完善（预计 1-2 周）

**目标**：完整工作流跑通

- 实时预览模式（全屏播放 + 页面切换）
- 导出面板 UI（分辨率、帧率、码率设置）
- FFmpeg 帧序列 → MP4 导出
- 导出进度与取消
- 全局快捷键
- 面板可拖拽调整
- 性能优化（纹理管理、内存释放）

**交付物**：完整功能版本，能从头到尾跑通创作 → 导出流程

## Phase 6：测试、打磨（持续）

**目标**：稳定性与体验提升

- 多图层性能压测
- 长视频导出稳定性
- AI 生成效果验收（Prompt 调优）
- 边界情况处理（损坏文件、中断恢复）
- 错误提示与日志

| 里程碑 | 产出 | 验收标准 |
