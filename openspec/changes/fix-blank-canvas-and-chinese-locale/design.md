## 背景

Phase 2 实现后，用户启动应用发现两个问题：
1. 画布区域显示空白——PixiJS 的 WebGL canvas 没有渲染出来，只有工具栏按钮可点击
2. 界面残留大量英文：属性面板标签（X/Y/Width/Height）、应用标题（TRPG Comic Studio）、控制台日志等

经代码审查，画布空白的根因很可能是 **CSS 百分比高度链在 PixiJS 初始化时尚未完成布局计算**，导致 `resizeTo: container` 创建了 0×0 的 canvas 元素。`centerArea` 虽然设置了 `backgroundColor: '#11111b'`，但 `canvas` 子元素撑开后可能覆盖了背景色。

## 目标 / 非目标

**目标：**
- PixiJS 画布在启动时可靠渲染，用户看到深色画布背景（#11111b）
- 若初始化失败，画布区域显示中文错误提示而非静默空白
- 所有用户可见文本使用中文
- 所有控制台日志使用中文

**非目标：**
- 优化 PixiJS 渲染性能
- 素材导入或画布内容填充（Phase 8 范畴）
- 国际化框架（i18n）——当前只做中文硬编码，不做多语言切换

## 决策

### 1. 画布容器：从百分比高度改为绝对定位

**决定：** 将 `canvas-container` div 的定位方式从 `width:100%; height:100%` 改为绝对定位 `position:absolute; inset:0`，父容器 `centerArea` 设为 `position:relative`。

**理由：** CSS 百分比高度依赖父元素有确定的 computed height。flex 布局中的 `flex:1` 虽然会计算高度，但 `height:100%` 在某些浏览器/时机下可能解析为 0（尤其是初始渲染帧）。绝对定位 `inset:0` 直接绑定到父元素的四边，不依赖百分比解析链，在任何时刻都能得到正确的非零尺寸。

**替代方案：** 使用 `ResizeObserver` 手动设置 canvas 尺寸。更复杂，且 PixiJS 的 `resizeTo` 本身就是基于 ResizeObserver 的，会造成双层监听。放弃。

### 2. 画布初始化失败兜底

**决定：** 在 `canvas-container` div 内部放置一个隐藏的兜底文字元素，仅在 PixiJS 初始化失败时显示。

具体做法：
- `canvas-container` div 内预先放一个 `<span>` 元素，`display:none`
- `initCanvasApp` 抛出异常时，`.catch()` 中将 canvas 容器背景设为 `#11111b`，显示错误提示文字

**为什么不用 React state？** 画布初始化是一次性的，不需要响应式更新。直接操作 DOM 避免额外渲染。

### 3. 中文化范围

**决定：** 所有面向用户的文本 + 所有 `console.*` 日志全部中文化。代码标识符（变量名、函数名、类型名）保持英文不变。

| 类别 | 英 → 中 | 示例 |
|------|---------|------|
| 属性标签 | X → X 坐标 | 属性面板字段 |
| 应用标题 | TRPG Comic Studio → TRPG 漫画工作室 | 工具栏 + HTML title |
| 占位文字 | 素材库 → 保持不变（已是中文） | 左侧面板 |
| 按钮文字 | background → 背景 | 已是中文，无需改动 |
| 控制台日志 | `[canvas-app]` → `[画布应用]` | console.warn/error |
| 注释 | 英文注释 → 中文注释 | JSDoc + 行内注释 |

**为什么不做 i18n 框架？** 当前是单人项目、中文用户、无国际化需求。引入 i18n 库会增加依赖和模板复杂度，收益为零。如果未来需要多语言，可以届时引入 `react-i18next` 并将硬编码中文替换为 `t('key')` 调用——改动范围可控。

### 4. 画布初始化日志增强

**决定：** `initCanvasApp` 成功后输出 `console.log('画布初始化成功：<宽>×<高>')`，帮助用户确认 WebGL 渲染正常。失败时输出 `console.error('画布初始化失败：<错误信息>')`。

**理由：** 之前只有 `.catch()` 中的日志，成功时没有任何输出。开发者无法判断 PixiJS 是否正确初始化。增加日志让问题排查有据可循。

## 风险 / 权衡

- **[风险] 绝对定位可能影响后续拖拽交互** → 画布容器从 flex 子项改为绝对定位后，不影响 PixiJS 内部的坐标系统。交互层（item-interaction）使用的是 PixiJS 的世界坐标，与 CSS 定位无关。验证项：确认 Ctrl+滚轮缩放和空格+拖拽平移仍然正常工作。
- **[权衡] 硬编码中文** → 当前是简单的最优解。未来如需多语言，需要全局搜索硬编码中文并替换为 i18n key。已在注释中标记所有用户可见文本位置，方便后续迁移。
